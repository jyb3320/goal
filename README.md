# 도장판 印 — 둘의 인생 운영실

둘이서 삶의 방향을 정하고, 12주 동안 실행하고, 정기적으로 솔직하게 복기하는 개인 인생 운영 시스템.

- **오늘** — 매일/기간 목표를 만들고 오늘·어제 도장을 찍는다. 친구 목표에는 리액션과 응원만 가능하다.
- **가장 큰 목표** — 각자 이루고 싶은 가장 큰 목표를 한 문장으로 적고 매일 확인한다.
- **인생 나침반** — 개인 헌법과 삶의 원칙을 적고 건강·일·돈·관계·성장 등 9개 영역의 현재 상태와 다음 행동을 점검한다.
- **12주 인생 시즌** — 이번 계절에 집중할 영역과 완료 기준, 하지 않을 것을 정한다. 프로젝트·루틴·해결할 문제를 시즌에 연결한다.
- **주간 인생 회의** — 실제로 일어난 일, 잘한 선택, 피한 문제, 걱정, 다음 주 약속을 각자 기록하고 함께 본다.
- **월간 방향 복기** — 실제 변화와 반복되는 패턴, 중단할 것과 다음 집중 방향을 기록한다.
- **결정 기록** — 중요한 결정을 내릴 당시의 사실·예상·두려움·이유를 남기고 나중에 실제 결과를 적는다.
- **파트너 원칙** — 평가보다 질문, 공격보다 정직, 실패보다 다음 선택을 우선한다.
- **콕 찌르기** — 친구를 콕 찌르면 상대 폰에 즉시 푸시가 간다 (횟수 제한 없음). 푸시를 못 받는 환경엔 앱 안 배너로 보인다.
- **실행 복기** — 어제 못 찍은 매일 목표는 소급해서 찍거나 왜 못 했는지 한 줄 남긴다. 이유는 기록 탭에 월별로 쌓인다 (180일 보관).
- **마을** — 도장이 쌓이면 XP·레벨이 오르고, 캔버스 마을에 식물이 자라고 캐릭터가 꾸며진다.
- **기록** — 월간 달력 히트맵 + 반성 노트로 지난 기록을 본다 (최근 1년 보관).

## 스택

React + Vite 프론트, Vercel 서버리스 함수(`api/`), Upstash Redis(상태 저장, 키 하나에 JSON + CAS 낙관적 잠금), Web Push 리마인더.

## 로컬 개발

```bash
npm install
npm run dev     # vite 대체 API 포함 (인메모리 — 재시작하면 초기화)
npm test        # vitest: 날짜/스트릭 로직 + 서버 인증/검증 테스트
```

`npm run dev`는 `vite.config.js`의 인메모리 미들웨어가 `/api/state`를 처리하므로 Redis 없이 동작한다. 실제 저장소로 테스트하려면 `vercel dev`를 쓰면 된다.

## 배포 (Vercel)

필요한 환경변수:

| 변수 | 설명 |
| --- | --- |
| `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` | Upstash Redis (Vercel Marketplace 연동 시 자동. 구 `KV_REST_API_*` 이름도 인식) |
| `VAPID_SUBJECT` | `mailto:본인이메일` (선택, 푸시 발신자 표기. 안 넣으면 기본값 사용) |
| `CRON_SECRET` | 설정하면 `/api/remind`를 크론 외 호출로부터 보호 (선택, Vercel이 자동으로 헤더에 넣어줌) |
| `AI_PROVIDER` | 우선 `nvidia` 사용 |
| `AI_BASE_URL` | NVIDIA NIM은 `https://integrate.api.nvidia.com/v1` |
| `AI_MODEL` | `moonshotai/kimi-k2.6` |
| `AI_API_KEY` | NVIDIA Build에서 발급한 API 키. 브라우저로 전달되지 않음 |
| `AI_ACCESS_PIN` | AI 기능을 보호하는 둘만의 PIN. 운영 환경에서는 설정 권장 |
| `AI_DAILY_LIMIT` | 사용자별 하루 AI 생성 횟수. 기본값 `5` |

웹 푸시용 VAPID 키는 **수동 발급이 필요 없다** — 첫 요청 때 서버가 스스로 키 쌍을 만들어 Redis에 저장하고 계속 재사용한다 (`api/_vapid.js`). Redis만 연결돼 있으면 배포만으로 끝.

`vercel.json`의 크론이 매일 12:00 UTC(= 한국 밤 9시)에 `/api/remind`를 호출해, 오늘 몫을 안 채운 사람에게 푸시를 보낸다. 앱에서 "🔔 밤 9시 알림 켜기"를 눌러 구독해야 받는다.

iOS는 홈 화면에 추가(PWA 설치)해야 푸시를 받을 수 있다.

## Kimi 인생 참모

`AI 참모` 탭에는 주간 참모, 목표 구조 설계, 정합성 검사, 회피 패턴, 결정 참모,
주간 회의 진행, 월간 보고 기능이 있다. 사용자가 선택한 기록 종류만 서버에서
요약해 Kimi로 보내며, 개인 보고서에는 본인 기록만 포함한다. 회의 진행 기능만
이미 서로에게 공개된 최근 주간 복기를 함께 사용한다.

- AI 응답이 인용한 기록 ID를 서버가 검증하고 실제 날짜와 원문을 함께 표시한다.
- 존재하지 않는 근거를 인용한 문장은 판단 보류로 대체한다.
- 목표 구조 초안은 사용자가 체크한 시즌·프로젝트·루틴만 등록한다.
- AI 응답은 자동으로 Redis에 저장하거나 친구에게 공개하지 않는다.
- 로컬 개발에서 `AI_API_KEY`가 없으면 실제 기록 구조를 사용한 개발용 예시 보고서를 보여준다.

## 인증

- 별도 로그인/비밀번호 없이 **이름만으로** 접속한다. 둘이서만 쓰는 걸 전제로 한 트레이드오프 — 링크와 상대 이름을 아는 사람은 그 사람 행세를 할 수 있다. 링크를 아는 두 사람만 쓴다면 문제 없다.
- 대신 서버는 소유권 검증은 계속한다: 내 목표에만 도장/기록/삭제 가능, 친구 목표에는 응원만 가능.
- 도장은 오늘/어제(서울 기준)만 찍을 수 있다 — API 직접 호출로도 소급 조작 불가.
- AI 엔드포인트에는 별도로 `AI_ACCESS_PIN`을 설정할 수 있고, 사용자별 일일 호출 제한이 적용된다.

## 데이터 보관

상태는 Redis 키 하나의 JSON. 무한히 커지지 않게 쓰기 때마다 컴팩션한다:

- 리액션: 14일 지나면 삭제 (UI는 최근 7일만 표시)
- 콕 찌르기: 오늘·어제 것만 (최근 20개)
- 반성 노트: 180일 (최근 300개)
- 도장: 400일 지나면 사용자별 집계(`archive`)로 합침 — XP와 마을 식물 수는 유지
- 기간 목표 진행 기록: 90일 지나면 목표당 한 건으로 합침 (합계 유지)
- 응원 메시지: 최근 50개
